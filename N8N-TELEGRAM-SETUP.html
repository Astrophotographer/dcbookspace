n8n + 텔레그램 알림 설정 가이드

장소사용신청서 신청 → n8n → 텔레그램 알림. 두 워크플로우(발송 + 자가 등록) 셋업.

목차

  1. 준비물
  2. 1단계: BotFather 로 봇 생성
  3. 2단계: n8n credentials 등록
  4. 3단계: 워크플로우 A — 발송용
  5. 4단계: 워크플로우 B — 자가 등록용
  6. 5단계: 환경 변수 설정 (Vercel · .env.local)
  7. 6단계: 검증
  8. 트러블슈팅
  9. 참고: 페이로드 스펙

0준비물

  • 텔레그램 계정 (관리자 본인 계정)
  • n8n 인스턴스 (self-hosted 또는 cloud — webhook 받을 수 있어야)
  • 장소사용신청서 관리자 권한 (Vercel/`.env.local` 수정 가능)

아키텍처: 수신자 목록 = Supabase telegram_subscribers 테이블이 SSoT. 사용자는 이름·휴대폰·소속 부서와 알림 범위(내 부서/모든 부서)를 등록하고, n8n 은 발송과 자가 등록 두 워크플로우만 담당.

1BotFather 로 봇 생성

  1. 텔레그램에서 @BotFather 검색 → 채팅 시작.
  2. /newbot 명령 입력.
  3. 봇 표시 이름 입력 (예: "등촌교회 알림")
  4. 봇 username 입력 — 반드시 bot 으로 끝나야 함. 예: PlaceUseRequestBot
  5. BotFather 가 답장하는 HTTP API token 을 안전한 곳에 복사 (n8n credential 에 등록할 값).
📸
BotFather /newbot 대화 캡쳐
봇 이름 입력 → username 입력 → token 받기까지 3컷

(선택) 봇 프로필 꾸미기

  • /setdescription — 봇 소개 글 (사용자가 처음 봇 채팅 열면 보임)
  • /setuserpic — 봇 프로필 사진
  • /setcommands — 명령어 메뉴 (없어도 됨, 자동 등록은 /start 만 씀)

주의: token 은 비밀번호급. 누가 알면 봇을 탈취당하니 GitHub · Slack 등에 절대 노출 금지.

2n8n credentials 등록

  1. n8n 좌측 메뉴 → Credentials → + Add credential
  2. 검색에서 Telegram API 선택
  3. 이름: 예) "DCB Telegram Bot"
  4. Access Token: 1단계에서 받은 BotFather token 붙여 넣기
  5. Save → 연결 테스트 성공 확인
📸
n8n Credentials → Telegram API 추가 화면

3워크플로우 A — 발송 (DCB Notify)

장소사용신청서가 신청 이벤트마다 호출. 페이로드의 recipients[] 배열을 each-loop 로 돌려 텔레그램 메시지 발송.

노드 구성

  1. Webhook — POST. URL 복사해 장소사용신청서의 WEBHOOK_TARGETS 에 등록.
  2. HMAC 검증 (Function 노드) — WEBHOOK_SECRET 으로 X-DCB-Signature 헤더 검증.
  3. Split In Batches 또는 Item Lists → Split Out — body.recipients 배열을 항목별로 분할.
  4. Set/Function — 이벤트 타입(x-dcb-event) 별 메시지 본문 템플릿.
  5. Telegram — chat_id = {{$json.chat_id}}, Text = 위 본문, Parse mode = MarkdownV2.

메시지 템플릿 예시 (Function 노드)

const event = $('Webhook').first().json.headers['x-dcb-event'];
const d = $('Webhook').first().json.body.data;
const r = $json;  // 현재 each-loop 의 recipient
const room = `${d.room.building_name} ${d.room.floor_label} ${d.room.name}`;
const titles = {
  'reservation.created':       '🆕 새 신청 접수',
  'series.created':            '🔁 정기 신청 접수',
  'reservation.step_approved': '✍️ 결재 단계 진행',
  'reservation.approved':      '✅ 결재 완료 (확정)',
  'reservation.rejected':      '❌ 반려',
  'reservation.cancelled':     '🚫 취소',
  'reservation.print_failed':  '🖨️ 인쇄 실패',
  'test.message':              '🔔 연결 테스트',
};
return [{
  json: {
    chat_id: r.chat_id,
    text: [
      `*${titles[event] ?? event}*`,
      `${r.name}님 알림입니다.`,
      ``,
      `📋 *${d.purpose ?? '연결 테스트'}*`,
      d.dept_name ? `🏷️ ${d.dept_name}` : null,
      d.room ? `📍 ${room}` : null,
      d.start_at ? `🕒 ${d.start_at} ~ ${d.end_at}` : null,
    ].filter(Boolean).join('\n'),
  }
}];

호환 모드: 페이로드에 recipients 가 없으면(=옛날 흐름) Switch 로 분기시켜 applicant.telegram_chat_id 하나만 보내도록 백업 경로 유지 가능. 단계적 마이그레이션 권장.

4워크플로우 B — 자가 등록 (DCB Link)

사용자가 /me/telegram 에서 이름·휴대폰·소속 부서와 알림 범위를 입력한 뒤 deep-link 를 누르면 봇이 /start TOKEN 메시지를 받음. 이 워크플로우가 받아서 장소사용신청서 사이트에 chat_id 등록.

노드 구성

  1. Telegram Trigger — Updates: message. 봇 메시지 수신 진입.
  2. IF — {{$json.message.text}} 가 /start 로 시작하는지 검사.
  3. Function — 메시지에서 토큰 + chat_id 추출: 
    const text = $json.message.text || '';
const token = text.replace(/^\/start\s+/, '').trim();
const chat_id = String($json.message.chat.id);
return [{ json: { token, chat_id } }];
  4. HTTP Request — 장소사용신청서 API 호출:
    • Method: POST
    • URL: {{ $env.SITE_URL }}/api/telegram/link
    • Headers:
      • Authorization: Bearer {{ $env.TELEGRAM_LINK_SECRET }}
      • Content-Type: application/json
    • Body: JSON { "token": "={{$json.token}}", "chat_id": "={{$json.chat_id}}" }
  5. Telegram (답장) — 성공 시: 
    chatId: {{ $('Function').first().json.chat_id }}
text: ✅ {{ $json.name }}님, {{ $json.scope_label }} 알림이 연결됐어요! 첫 메시지를 곧 보내드릴게요.
  6. 오류 분기 — HTTP 4xx/5xx 시 사용자에게 안내 ("토큰 만료" 등). HTTP Request 노드의 Continue On Fail 켜고 IF 로 분기.
📸
n8n 워크플로우 B 캔버스

중요: 봇이 메시지 받으려면 텔레그램 측에서 webhook 또는 polling 모드 활성. n8n Telegram Trigger 는 자동으로 webhook 등록하니 워크플로우 활성화만 하면 됩니다.

워크플로우 B JSON 샘플 (복붙용) 
{
  "name": "DCB Link",
  "nodes": [
    {
      "parameters": { "updates": ["message"] },
      "name": "Telegram Trigger",
      "type": "n8n-nodes-base.telegramTrigger",
      "typeVersion": 1.1,
      "credentials": { "telegramApi": { "name": "DCB Telegram Bot" } }
    },
    {
      "parameters": {
        "conditions": {
          "string": [
            { "value1": "={{$json.message.text}}", "operation": "startsWith", "value2": "/start " }
          ]
        }
      },
      "name": "Is /start?",
      "type": "n8n-nodes-base.if",
      "typeVersion": 1
    },
    {
      "parameters": {
        "functionCode": "const text=$json.message.text||'';\nconst token=text.replace(/^\\/start\\s+/, '').trim();\nconst chat_id=String($json.message.chat.id);\nreturn [{ json: { token, chat_id } }];"
      },
      "name": "Parse Token",
      "type": "n8n-nodes-base.function",
      "typeVersion": 1
    },
    {
      "parameters": {
        "url": "={{$env.SITE_URL}}/api/telegram/link",
        "requestMethod": "POST",
        "jsonParameters": true,
        "options": { "responseFormat": "json" },
        "headerParametersJson": "{\"Authorization\": \"Bearer \" + $env.TELEGRAM_LINK_SECRET}",
        "bodyParametersJson": "={ \"token\": $json.token, \"chat_id\": $json.chat_id }"
      },
      "name": "POST /api/telegram/link",
      "type": "n8n-nodes-base.httpRequest",
      "typeVersion": 3
    },
    {
      "parameters": {
        "chatId": "={{ $('Parse Token').first().json.chat_id }}",
        "text": "=✅ {{ $json.name }}님, {{ $json.scope_label }} 알림이 연결됐어요! 첫 메시지를 곧 보내드릴게요."
      },
      "name": "Reply Success",
      "type": "n8n-nodes-base.telegram",
      "typeVersion": 1.1,
      "credentials": { "telegramApi": { "name": "DCB Telegram Bot" } }
    }
  ]
}

5환경 변수 설정

Vercel 프로젝트 + 로컬 .env.local 양쪽 모두.

변수위치설명
NEXT_PUBLIC_TELEGRAM_BOT_USERNAMEVercel + 로컬BotFather username (@ 빼고). 비면 자동 흐름 비활성.
TELEGRAM_LINK_SECRETVercel + 로컬 + n8n32자 임의 문자열. openssl rand -hex 32
WEBHOOK_TARGETSVercel + 로컬n8n 워크플로우 A 의 Webhook URL (콤마 구분 여러 개 가능)
WEBHOOK_SECRETVercel + 로컬 + n8n16자 이상. HMAC 검증용. 양쪽 동일해야.
SITE_URL (n8n 내)n8n env장소사용신청서 호스트. 예: https://dcbookspace.vercel.app

n8n self-hosted 의 경우: env 는 docker-compose 또는 systemd 서비스의 environment 섹션에 추가. 변경 후 n8n 재시작 필요.

6검증

자동 흐름 (모바일 권장)

  1. /me/telegram 접속
  2. 본인 정보 입력 → 이름·휴대폰·소속 대분류/중분류
  3. 알림 범위 선택 → 내 부서만 또는 모든 부서
  4. "텔레그램으로 자동 연결" 큰 버튼 클릭
  5. 텔레그램 앱이 열리며 /start TOKEN 자동 전송
  6. 봇이 "✅ 연결됐어요" 답장
  7. 웹 화면이 "✅ 등록 완료" 로 전환

수동 흐름

  1. /me/telegram → 수동 입력 탭
  2. chat_id 입력 (@userinfobot 으로 확인 가능)
  3. "등록하고 테스트 메시지 받기" 클릭 → 텔레그램 도착 확인

실 신청 알림

  1. 등록 후 새 신청서를 만들어 본다
  2. 봇이 같은 chat_id 로 "🆕 새 신청 접수" 알림 발송하는지 확인
  3. 결재 완료 시 "✅ 결재 완료 (확정)" 알림 확인

성공 신호: Supabase telegram_subscribers 테이블에 row 들어가고, n8n 실행 로그(Executions) 에 status=success 가 찍히면 정상.

!트러블슈팅

봇이 답장 안 함

  • n8n 워크플로우 B 가 Active 상태인지 확인
  • n8n 실행 로그 (Executions) 에 trigger 가 찍히는지 확인
  • BotFather 에서 /setprivacy → DISABLE — 그룹에서 봇이 메시지 읽을 수 있도록 (1:1 채팅엔 영향 없음)

HTTP 401 unauthorized

  • n8n env TELEGRAM_LINK_SECRET 와 Vercel env 값이 정확히 같은지 비교
  • n8n HTTP Request 노드의 Authorization 헤더에 Bearer 접두사 포함됐는지 (스페이스 1개)

HTTP 410 token_expired

  • 토큰 발급 후 10분 안에 시작 안 함 → 사이트로 돌아가 새로 발급

발송이 안 됨 (메시지 도착 X)

  • n8n 실행 로그에서 워크플로우 A 가 트리거됐는지 확인
  • recipients[] 가 비어있다면 → Supabase RPC get_telegram_recipients 가 매칭 안 함. 구독자의 active, 이벤트 종류, 부서 필터 확인.
  • Telegram 노드 실패 시 → chat_id 형식 (개인 양수, 그룹 -100…) 확인. 봇이 그룹에서 메시지 보내려면 그룹 멤버여야.

HMAC 검증 실패 (Webhook 401)

  • WEBHOOK_SECRET 양쪽 정확히 같은지
  • n8n Function 노드의 검증 코드가 raw body 를 사용하는지 (JSON 파싱 전 원본)

i참고: 페이로드 스펙

장소사용신청서 → n8n 워크플로우 A 가 받는 POST body:

{
  "event": "reservation.created",       // 또는 series.created · step_approved · approved · rejected · cancelled · print_failed · test.message
  "delivery": "uuid",                   // 멱등키
  "occurred_at": "2026-05-13T10:00:00.000Z",
  "data": {
    "kind": "reservation",              // 또는 "series" · "test"
    "id": "...",
    "ref_no": "...",
    "status": "pending",                // pending · approved · rejected · cancelled
    "start_at": "...", "end_at": "...", // reservation
    "start_date": "...", "end_date": "...", "weekday": 0,  // series
    "purpose": "...",
    "applicant": { "name": "...", "phone": "...", "telegram_chat_id": null },
    "dept_name": "...",
    "room": { "name": "...", "floor_label": "...", "building_name": "..." },
    "next_approvers": [ /* step_approved 시에만 */ ],
    "recipients": [
      { "chat_id": "123456789", "name": "홍길동", "dept_name": "모든 부서" },
      ...
    ]
  }
}

각 요청 헤더:

  • X-DCB-Event: reservation.created
  • X-DCB-Delivery: <uuid>
  • X-DCB-Signature: sha256=<base64url HMAC> (WEBHOOK_SECRET 설정 시)
