# UI 컴포넌트 규칙 — 노년층 친화

이 프로젝트의 주 사용자는 **장로·차장·당회장**(어르신). UX 결정은 거의 다 이쪽으로 기울임.

## 글자 크기
- 루트 폰트는 **17px** (Tailwind 기본 16px 아님). 작은 텍스트 추가 시 base 크기 깨지 않게 주의
- 입력칸은 크게. 모바일 터치 타겟 44px 이상

## 색맹 대응 — 다중 신호 원칙
- 상태 표시는 **색만으로 끝내지 말 것**. 항상 색 + 아이콘 + 텍스트 + (필요 시) 그림자/테두리 다중 신호
- 예: PIN 입력칸 활성/비활성, 호실 점유 상태(비어있음/진행중/완료/혼합), 결재 단계 등

## 입력 자동 정렬
- 휴대폰: 8자리 입력 시 `010-1234-5678` 자동
- 날짜: 8자리 입력 시 dash 자동 + 월/일 범위 자동 보정 (윤년 반영)
- 포커스·커서 위치는 사용자가 손대지 않게 자연스럽게 유지

## 오류 피드백
- PIN 오답: **0.45초 흔들림 + 빨간 강조 + 즉시 비움**. 토스트만 띄우는 식으로 바꾸지 말 것 (어르신은 모달/토스트를 놓치는 경우가 많음)

## 기본 원칙
- 새 인터랙션 추가 시 "마우스 못 쓰는 70대가 처음 봐도 되나?" 기준으로 검토
- 모달은 최소화. 꼭 필요할 땐 큰 버튼 + 명확한 라벨 (확인/취소가 아니라 "결재 취소" 같이 동작 그대로)
